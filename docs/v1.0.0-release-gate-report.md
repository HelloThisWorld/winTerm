## Verified Locally

- The x64 Release unpackaged application built successfully.
- The application stage passed branding, identity, architecture, payload, and
  secret/debug-output inspection across 170 files.
- `winTerm-1.0.0-setup-x64.exe` compiled with pinned Inno Setup 6.7.3 and
  passed installer metadata and layout verification.
- `winTerm-1.0.0-portable-x64.zip` passed archive and 172-file Portable layout
  verification.
- The Portable archive launched from a temporary extraction, created its
  adjacent `data\settings.json`, and left installation state unchanged.
- The current-user installer passed install, default-profile launch, Windows
  PowerShell launch, Command Prompt launch, upgrade/repair, uninstall,
  reinstall, registration cleanup, user-data preservation, and Microsoft
  Windows Terminal isolation checks.
- Version, branding, release-workflow, pane-control source, and diff checks
  passed.
- The native Terminal app library and Settings Model unit-test library built
  in x64 Debug; all 65 tests selected by `*WinTerm*` passed.

The installer is intentionally unsigned. Windows can display Unknown Publisher
or a SmartScreen warning. The GitHub Release workflow records this status in
the release metadata and verifies the public assets after downloading them.

## Known Limitations

- Live pointer UI automation for pane-handle docking has not been completed.
- Narrator, High Contrast, and mixed-DPI pane-docking scenarios have not been
  completed as live end-to-end tests.
- New Window live transfer remains disabled until same-process ownership
  transfer and rollback are verified end to end.
- The unfiltered upstream Settings Model suite is not a local release pass: its
  TAEF host terminated with `0xC0000409` in unrelated profile/media-resource
  tests (144 passed, 64 failed). The winTerm-specific selection passed.

These limitations are documented in the v1.0.0 release notes. Publication was
explicitly authorized with these limitations after the release artifacts and
supported local acceptance gates passed.

## GitHub Publication Gate

The `v1.0.0` tag workflow must still complete all server-side gates before
publication. It verifies the exact tag and clean checkout, runs smoke and
compiled Release tests, repeats current-user and all-users installer tests,
repeats Portable launch isolation, generates checksums and SBOMs, creates
artifact attestations, tests every downloaded Draft asset, publishes the stable
Release, and then downloads and tests every public asset again.

The public download URLs are:

- `https://github.com/HelloThisWorld/winTerm/releases/download/v1.0.0/winTerm-1.0.0-setup-x64.exe`
- `https://github.com/HelloThisWorld/winTerm/releases/download/v1.0.0/winTerm-1.0.0-portable-x64.zip`

Release authorized with documented non-blocking limitations.

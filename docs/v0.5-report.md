# winTerm v0.5 Visual Docking report

## Verdict

**Not accepted for runtime use.** The model, persistence, transaction, ownership, security, preview, keyboard, diagnostics, and test foundations are implemented. The XAML drag/drop host adapter is intentionally feature-disabled because this environment cannot build or launch winTerm, run compiled rollback tests, or verify live session movement. `DockingSettingsModel::enableRuntimeDocking` defaults to `false`.

No shell restart, live-session duplication, cross-process transfer, Terminal Broker, or post-exit persistence fallback was added.

## Security results

- Drag payloads use 256-bit BCrypt-generated opaque tokens.
- Tokens are internal, expiring, source-instance-bound, capability-bound, and single-use.
- Forged, expired, wrong-instance, and consumed tokens are rejected by the source model.
- Cross-process transfer is rejected.
- Diagnostics use an allow-list of error categories and a bounded in-memory ring.
- Workspace import retains executable, command, URL, oversized-file, and unknown-node defenses.
- Drag state, tokens, leases, terminal output, command text, clipboard content, environment, and full working directory are not persisted or logged.
- Session ownership validation rejects duplicate ownership in the model.

Compiled security tests and adversarial runtime drag tests remain unavailable.

## Known limitations

- The XAML pointer adapter, overlay renderer, preview adorner, tab-strip insertion indicator, drag handles, focus adapter, command-palette registration, and Settings page are not connected.
- Runtime docking remains feature-disabled.
- Cross-process tab and pane transfer is unsupported.
- Same-process live transfer is not compiled or manually verified.
- Unsupported session types have model-level capability rejection but no runtime matrix.
- Mixed-DPI monitor edges, negative coordinates, portrait displays, monitor removal, and new-window placement are not runtime tested.
- Alternate buffer, mouse reporting, search, selection, read-only, and broadcast-input modes are not runtime tested.
- Narrator, High Contrast rendering, localization, and 100%–300% XAML scaling are not runtime audited.
- Undo/redo is in memory and requires referenced live sessions to remain available.
- Maximum supported panes remain 64 per tab and 512 per Workspace.
- Corner docking intentionally creates one Empty Slot; it does not create an L-shaped terminal pane.
- Schema 2 readers reject unknown layout nodes. Older readers are not expected to understand Empty Slots.
- Current-runtime diagnostics require a future adapter to publish a redacted runtime layout snapshot.

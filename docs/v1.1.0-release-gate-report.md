# winTerm 1.1.0 release-gate report

Date: 2026-07-23

Decision: **winTerm 1.1.0 blocked before release**

The native implementation, Release build, live two-pane resize path, common
snap ratios, Alt bypass, Escape rollback, and x64 artifacts are complete.
Publication remains blocked because the full nested/DPI/accessibility/rendering/
shell/performance/upgrade matrix and a privacy-safe visual comparison with the
website reference have not been completed.

## Removed functionality

- Removed winTerm pane-header drag repositioning and the hidden experimental
  movement path.
- Removed pane docking overlays, targets, previews, drag sessions, session
  leases, movement actions, movement keyboard mode, obsolete settings, and
  their dedicated scripts/tests/docs.
- Removed winTerm move-pane command-line parsing and movement menu entries while
  retaining upstream code behind the non-winTerm branding boundary.
- Retained top-level tab behavior and directed Top, Bottom, Left, and Right
  splitting.
- Existing settings/workspaces safely ignore obsolete movement fields; schema
  version remains 2.

## Resize implementation

- A 1-DIP visual divider is separate from a 12-DIP logical pointer target.
  At the tested 144-DPI scale, UI Automation measured the expected 18-pixel
  physical target.
- Pointer press captures the owning split ID, original ratio, local geometry,
  minimum sizes, and valid snap points.
- Pointer movement updates only runtime layout and terminal dimensions.
  Serialization and disk writes are outside the move path.
- Pointer release commits one normalized ratio and raises one history record.
- Escape/cancellation restores the original ratio and creates no commit.
- Minimum sizes include pane-header height and terminal constraints.
- Split-node IDs and divider visuals survive runtime split, close, swap, and
  zoom tree reconstruction paths.
- The real x64 Portable Release horizontal divider changed by 216 physical
  pixels during a controlled drag and retained the committed position.
- Nested/local ownership passes model tests. A live nested automation attempt
  was inconclusive because its QA window closed without a recorded Application
  crash; it is not counted as manual acceptance.

## Snapping

- Default ratios: 25%, 33.333%, 50%, 66.667%, and 75%.
- Threshold: 8 logical pixels.
- Release threshold: 14 logical pixels.
- Hysteresis keeps an active snap until the larger release threshold is crossed.
- Alt bypasses snapping.
- The indicator presents `1 / 4`, `1 / 3`, `50 / 50`, `2 / 3`, or `3 / 4`
  without taking separate focus; the divider HelpText provides a non-color
  signal.
- Ratios that violate either child's local minimum size are filtered out.
- Live Release evidence passed all five common ratios at exact measured
  positions. Alt produced a free position four physical pixels from the 50%
  target, and Escape restored the exact pre-drag position.

## UI refactor

- Window shell/title bar use the compact native Windows frame and centralized
  dark shell resources.
- Tabs use a compact dark surface, profile icon, active-state contrast, close,
  plus, and profile controls.
- Pane headers use a pane/profile icon, title, real active status, and overflow
  button. No grip cursor or pane-movement accessible name remains.
- Dividers use low-contrast idle treatment and mint active/snap treatment.
- The unified pane menu retains directed split, zoom, Balance Panes, settings,
  and close without movement actions.
- Settings expose Application UI options and Docking and layout / Pane Resizing
  options.
- User profile terminal color schemes remain independent from UI-shell tokens.

## Workspace and history

- Final ratios continue through the existing split-layout persistence format;
  no schema increment was required.
- Transient pointer, snap, modifier, and indicator state is not serializable.
- One continuous resize creates one exact undo/redo record; cancel creates none,
  and a new commit clears redo.
- Balance Panes uses the immediate owner and participates in the same history
  path.
- Workspace model, validation, restore-planning, recovery, and import-security
  tests pass.
- A real close/reopen ratio-restore scenario and a 1.0.x upgrade round trip were
  not executed, so those release gates remain open.

## Accessibility

- The real divider is exposed as an enabled, focusable Thumb named
  `Resize top and bottom panes`.
- Pointer press moved keyboard focus to that divider.
- Snap HelpText announced the active ratio and Alt alternative.
- Pane icon and overflow controls expose accurate pane/menu names.
- Keyboard resize actions use 2% normal and 5% accelerated steps in the model.
- High Contrast fallback and reduced-motion-aware resources are implemented.
- Full Narrator, keyboard-menu, High Contrast, and reduced-motion manual passes
  were not run.

## DPI

- The live resize, five snap points, Alt bypass, Escape rollback, target
  alignment, and indicator placement passed at 144 DPI (150%).
- Model tests cover logical-pixel DPI invariance.
- 100%, 125%, 200%, mixed-DPI, portrait, negative-coordinate, and
  monitor-transition manual cases remain open.

## Performance

- Pointer movement performs local ratio evaluation and row/column updates only.
  It does not enumerate profiles, reload themes, serialize workspaces, write to
  disk, reconstruct the window, or access the network.
- Model tests pass 500 repeated resize transactions.
- Frame time, terminal resize latency, memory growth, continuous CPU, large
  scrollback, rapid output, 4-/20-pane, and deeply nested measurements were not
  collected. No performance acceptance claim is made.

## Testing

Passed:

- Full Release x64 build with package generation and all requested unit-test
  projects.
- Final `Relevant` Release suite, including version/branding, scripts,
  workspace foundations, privacy, pane source boundaries, Settings Model,
  Terminal App, and Control tests.
- Control unit tests: 30 passed, 0 failed.
- Pane-resize source boundaries: snapping, cancellation, DPI, and history.
- Real Portable Release startup at 150% DPI.
- Real directed horizontal split, pane headers, pane overflow menu, Docking and
  layout settings, and Application UI settings.
- Real two-pane horizontal drag, all five common ratios, Alt bypass, and Escape
  rollback.
- Portable layout/identity/isolation verification.
- Installer layout/identity/signature-state verification.
- `git grep -n -i "kaname"` through the repository branding gate: no matches in
  winTerm-owned tracked files.

Not completed:

- Live vertical and nested divider acceptance, 4-/20-pane and deep layouts.
- Rendering matrix: ANSI 16/256/True Color, CJK, emoji, box drawing, cursor,
  selection, alternate screen, and rapid output while dragging.
- Full PowerShell, Windows PowerShell, CMD, and WSL regression matrix.
- Workspace close/reopen persistence and 1.0.x upgrade.
- Installer install/uninstall round trips.
- Narrator, High Contrast, reduced motion, and complete DPI/multi-monitor matrix.
- Performance measurements.

## Visual parity

The website reference was inspected read-only. Its source uses the same
blue-black shell, near-black terminal surface, compact tabs and headers, thin
blue-grey borders, muted text, and mint accent (`#63e6be`) reproduced by the
native token system.

The real Release build was inspected through non-content UI Automation and
controlled interactions. The required title/tab/pane/menu/settings structure is
present. A content-bearing screenshot was not captured because the desktop
privacy safeguard rejected it, so pixel/material visual parity is not claimed.

The website currently uses synthetic HTML/CSS product compositions and still
contains grip-style pane imagery/copy. The post-release changes are recorded in
`docs/design/v1.1-website-handoff.md`; the website remains intentionally
unchanged in this task.

## Final decision

**winTerm 1.1.0 blocked before release**

Do not publish or tag v1.1.0 until every open release gate in
`docs/v1.1-acceptance.md` has objective passing evidence.

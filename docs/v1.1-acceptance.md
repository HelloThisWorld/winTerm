# winTerm 1.1 acceptance

This document distinguishes automated source/model evidence from validation
that requires a real Release application. A checked source item is not a claim
that native UI, renderer, shell, DPI, installer, or upgrade scenarios passed.

## Automated implementation coverage

- [x] Pane movement defaults, menus, overlays, sources, transactions, settings,
  keyboard mode, and dedicated tests removed or disabled for winTerm.
- [x] Directed Top, Bottom, Left, and Right split entry points retained.
- [x] Separate 1-pixel divider and 12-pixel logical hit target implemented.
- [x] Immediate-split geometry, minimum-size filtering, common ratios, Alt
  bypass, and 8/14-pixel hysteresis implemented.
- [x] Escape/capture-loss rollback and one-record commit model implemented.
- [x] Exact undo/redo and Balance Panes history path implemented.
- [x] Final ratios continue through existing startup-action persistence;
  transient resize state is outside serialization.
- [x] Settings model and native editor controls added.
- [x] Centralized native design tokens and High Contrast divider fallback added.
- [x] Model/source tests cover common ratios, invalid ratios, custom input, Alt,
  hysteresis, cancel, local geometry, logical DPI conversion, history, redo
  clearing, and 500 repeated resize transactions.

## Release-build and manual gates

- [x] Release x64 application, MSIX, and Settings Model, Terminal App, and
  Control unit-test projects build.
- [x] A clean Portable Release launch at 144 DPI (150%) remained responsive,
  accepted a directed horizontal split, and exposed a 12-DIP divider hit
  target as an 18-pixel UI Automation element.
- [x] A real two-pane horizontal divider updated continuously and committed a
  new ratio. UI Automation measured a 216-pixel divider change.
- [x] The real Release divider snapped to 25%, 33.333%, 50%, 66.667%, and 75%,
  with the correct accessible ratio text at every point.
- [x] Holding Alt produced an unsnapped ratio four pixels from the 50% snap
  point.
- [x] Pressing Escape during a real drag restored the exact original divider
  position.
- [ ] Divider behavior validated in vertical, nested, 4-, and 20-pane layouts.
  A nested automation attempt was inconclusive and is not counted as a pass.
- [ ] Heavy output, large scrollback, alternate screen, mouse reporting, and
  terminal resize latency validated.
- [ ] ANSI 16/256/True Color, CJK, emoji, box drawing, cursor, and selection
  rendering validated.
- [ ] PowerShell, Windows PowerShell, CMD, and WSL launch/regression pass.
- [x] The real divider exposes an accessible Thumb name, keyboard focus, and a
  non-color snap-state HelpText signal; pane icons and overflow buttons expose
  accurate names.
- [ ] Narrator, keyboard menus, High Contrast, and reduced-motion pass.
- [x] The real two-pane drag and snap sequence passes at 150% DPI.
- [ ] 100%, 125%, 200%, mixed-DPI, portrait, negative-coordinate, and
  monitor-transition scenarios pass.
- [x] Real Release structure inspected: compact tabs, two pane headers, pane
  icons, overflow menus, Docking and layout settings, and Application UI
  settings are present and responsive.
- [ ] Release application visually compared with the website reference.
- [x] x64 installer and Portable ZIP build; static installer and Portable
  identity/layout/isolation verification passes; the Portable build launches.
- [ ] Installer install/uninstall round trips and upgrade from 1.0.x preserve
  settings and workspaces.
- [x] Signature status, byte size, and final SHA-256 values are recorded in
  `docs/v1.1.0-release-gate-report.md`.

Until every blocking manual and packaging gate is complete, the release
decision is **winTerm 1.1.0 blocked before release**.

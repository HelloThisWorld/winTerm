# Migrating to winTerm 1.1 pane resizing

winTerm 1.1 removes pane drag repositioning and replaces its primary pointer
interaction with border resizing. Existing shells, profiles, tabs, split
trees, final split ratios, workspaces, themes, and application data remain
compatible.

## Removed settings

The application no longer reads or exposes winTerm pane movement fields such
as `paneDragEnabled`, `paneTearOutEnabled`, `cornerPaneDockingEnabled`,
`paneMoveMode`, `paneDockTarget`, or `paneDragHistory`. Unknown legacy fields
are ignored on load; they are not converted into another movement path and do
not trigger repeated writes. Tab dragging and tab tear-out settings are
unrelated and remain available.

Legacy `movePane` actions remain parse-safe so an old settings file can load,
but the winTerm action is disabled, absent from defaults and menus, and its
runtime pane-move implementation is not compiled into the winTerm build.
Remove obsolete custom keybindings when convenient.

## New settings

Settings → Docking and layout → Pane resizing provides:

- enable resize snapping;
- Balanced, Common ratios, or Custom snap points;
- custom comma-separated ratios;
- ratio indicator;
- Alt bypass;
- advanced logical-pixel snap threshold;
- reset to defaults.

Settings → Appearance → Application UI provides density and pane-header
presentation controls.

## Persistence

Workspace schema remains version 2. A final ratio uses the established split
serialization field, so no new persistent state is needed. Hover, pointer,
candidate snap, indicator, modifier, original ratio, and history cursor state
are never serialized. Opening and saving a 1.0 workspace preserves its safe
unknown fields and final layout data.

# winTerm 1.1 website handoff

The official website was a read-only design reference for the native 1.1 work.
Do not update it until the real Release build passes native visual acceptance.

After acceptance:

1. Remove screenshots or copy that present pane repositioning by header or grip
   drag.
2. Replace any grip-style pane symbol that suggests movement with the real
   pane/profile icon from the accepted build.
3. Describe border resizing, common-ratio snapping, Alt bypass, directional
   splits, and Balance Panes.
4. Capture a sanitized real 1.1.0 screenshot snapped at 50/50.
5. Capture the real directional split menu with Top, Bottom, Left, and Right.
6. Use the accepted native title bar, compact tabs, pane headers, dividers, and
   terminal surfaces; do not recreate them as fictional website-only controls.
7. Preserve the website's current blue-black, thin-border, mint-accent visual
   style.

Screenshots must contain no usernames, machine names, file paths, tokens,
private commands, or other personal information. Record the exact 1.1.0 build
commit used for capture.

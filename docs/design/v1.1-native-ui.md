# winTerm 1.1 native UI

The 1.1 application shell translates the official website product imagery into
native XAML and C++ controls. It is a dense desktop utility, not a browser
dashboard.

## Window and tabs

The compact title bar keeps the original winTerm title and artwork while the
Windows host retains caption buttons, drag regions, the system menu, double
click maximize, resize borders, and Snap Layouts. The tab strip uses a
blue-black surface, compact tabs, ellipsized titles, profile icons, close,
plus, and profile-dropdown controls, subtle hover contrast, and thin
separators. Existing tab creation, closing, reorder, keyboard navigation, and
overflow behavior remain intact.

## Panes

Split layouts use near-black terminal surfaces and 27-pixel compact pane
headers by default. Each header contains a terminal/pane icon, title, real
active/read-only/running status where applicable, and a three-dot overflow
button. Clicking the icon focuses the pane; right-click uses the unified pane
menu. It has no grip cursor, movement accessible name, or repositioning state.

Idle dividers are thin desaturated blue-grey, hover is brighter, and active
resize uses mint. The pointer target is wider than the line. The pane menu keeps
directional splitting, zoom, Balance Panes, settings, and close without
movement operations.

## Tokens and customization

Colors, typography, spacing, radii, shadows, motion, density, and interactions
are centralized under `src/winterm/Design`. Settings → Appearance provides the
Application UI theme, compact/comfortable density, pane-header visibility,
profile icon, active status, and animation preference. User terminal profile
themes remain authoritative inside terminal content.

High Contrast switches interactive dividers to theme brushes. The new shell
does not depend on animation, so reduced-motion behavior remains stable.

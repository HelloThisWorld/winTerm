# winTerm v0.6 Public Beta feature freeze

## Release scope

v0.6 is a stabilization release. It does not add a terminal broker, persistent sessions after exit, cloud synchronization, AI features, plugins, credentials storage, remote persistence, or a theme marketplace. The release target is `0.6.0-beta.1` with package version `0.6.0.0`.

## Feature classification

| Feature | Status | Default | Public Beta note |
| --- | --- | --- | --- |
| Workspace Restore and Named Workspaces | Beta | On | Source and fixture validation are covered; installation and live recovery still require Windows manual verification. |
| Edge and Corner Docking | Disabled | Off | The model, previews, rollback, and keyboard path are covered, but the XAML adapter remains disabled until live transfer validation completes. |
| Empty Slots and Layout Undo/Redo | Beta | On in the model | Covered by source-level validation and fixtures. |
| Cross-window tab transfer | Experimental | Off | Only same-process transfer is in scope; no shell restart fallback is used. |
| Cross-window pane transfer | Disabled | Off | Cross-process transfer is rejected. |
| Theme import and app-private fonts | Beta | On | Asset validation is automated; runtime rendering still needs manual verification. |
| Linux Safe Compatibility | Beta | On | Conservative mappings apply only to local shells. No general Linux translation is provided. |
| PowerShell and CMD integration | Beta | On | Automated asset and safety checks are available. |
| ARM64 | Not supported | N/A | No ARM64 package has been built or launched for this beta candidate. |

## Release blockers

P0 blocks a Public Beta: data loss, workspace corruption, unsafe import execution, unexpected shell termination during a docking transaction, package identity collision, secret leakage, or a package that cannot launch.

P1 must be fixed before an RC unless the release report records the user impact, workaround, target version, and explicit acceptance. P2 and P3 issues may ship only when their user impact is documented.

## Feature flags

`DockingSettingsModel::enableRuntimeDocking` remains `false`. This is intentional: model-level work must not enable unverified live session movement. The feature will be reconsidered only after a provisioned Windows environment verifies rollback, multi-window behavior, and accessibility.
